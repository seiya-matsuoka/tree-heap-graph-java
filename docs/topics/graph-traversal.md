# グラフの走査

## 1. 概要

グラフの走査は、頂点同士のつながりをたどりながら、頂点を順番に訪問する処理。  
グラフでは枝分かれや閉路があるため、**訪問済み管理** が重要になる。

今回のトピックでは、次の内容を確認する。

- グラフに対する DFS
- グラフに対する BFS
- 到達可能性
- 連結性
- 連結成分

## 2. 基本の動き

グラフの走査では、同じグラフでも DFS と BFS で訪問順が変わる。  
DFS は深く進み、BFS は近い頂点から順に広げる。

このトピックでは、次の流れでグラフの走査を確認する。

1. 入力辺からグラフを作る
2. 開始頂点から DFS を行う
3. 開始頂点から BFS を行う
4. target へ到達できるかを確認する
5. グラフ全体が連結かを確認する
6. 連結成分のまとまりを確認する

## 3. 処理の流れ

このトピックでは、次の順で走査結果を確認する。

1. 隣接リストを構築する
2. 開始頂点から DFS を行う
3. 開始頂点から BFS を行う
4. BFS の結果を使って target への到達可否を確認する
5. 全頂点を対象に連結成分を集める
6. 連結成分数から連結性を確認する

trace あり実行では、どの頂点を訪問したか、どの頂点をキューや再帰の次候補にしたかも確認できる。  
そのため、DFS と BFS の進み方の違いを出力から追える。

## 4. 計算量

### DFS

- 時間計算量: O(V + E)
- 空間計算量: O(V)

### BFS

- 時間計算量: O(V + E)
- 空間計算量: O(V)

### 到達可能性・連結性の確認

- 時間計算量: O(V + E)
- 空間計算量: O(V)

頂点数を V、辺数を E としたとき、基本的には各頂点と各辺を一度ずつ確認する処理になる。

## 5. メリット / デメリット

### メリット

- DFS と BFS の違いを通して、探索の進み方の違いを確認できる
- target に到達できるかを調べられる
- グラフ全体がつながっているかを確認できる
- 連結成分ごとのまとまりを把握できる

### デメリット

- 訪問済み管理をしないと、同じ頂点を繰り返してしまう
- 木より構造が複雑で、出力を追いづらい
- DFS / BFS / 連結成分の処理が混ざると理解しづらくなりやすい

## 6. Java実装のポイント

- 隣接リストは `List<List<Integer>>` で表現する
- DFS では再帰を使って深く進む
- BFS では `Queue<Integer>` を使って近い頂点から順に進む
- 訪問済み管理は `boolean[] visited` で行う
- 連結成分の確認では、未訪問頂点から探索を始める処理を繰り返す

## 7. コアとなる処理・重要なコードの見どころ

グラフの走査で重要なのは、**訪問済みを管理しながら DFS / BFS を進める処理** と、**複数の連結成分を集める処理**。

### DFS

DFS では、現在の頂点を訪問済みにし、隣接頂点へ再帰的に進む。

```java
visited[currentVertex] = true;
order.add(currentVertex);
```

ここで見たいのは、今いる頂点を記録してから、まだ訪問していない隣接頂点へ深く進む点。  
木の DFS と似ているが、グラフでは閉路があるため訪問済み管理が必要になる。

### BFS

BFS ではキューを使い、開始頂点に近い順に訪問する。

```java
queue.add(startVertex);
visited[startVertex] = true;
```

その後、キューから取り出した頂点の隣接頂点を確認し、未訪問ならキューへ追加する。  
これにより、距離が近い頂点から順に見る流れになる。

### 連結成分を集める処理

連結成分の確認では、まだ訪問していない頂点を見つけたら、その頂点から探索を始めて 1 つの成分を集める。

```java
if (visited[startVertex]) {
    continue;
}
```

この処理を全頂点に対して繰り返すことで、グラフ全体が何個の成分に分かれているかを確認できる。

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic graph-traversal --trace
```

### 任意入力

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic graph-traversal --input 0-1,0-2,1-3,2-4,4-5 --target 5 --trace
```

### size 指定

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic graph-traversal --size 10 --target 9
```

### 出力で確認する内容

- 入力辺
- 開始頂点
- target
- DFS の訪問順
- BFS の訪問順
- 到達可能性
- 連結性
- 連結成分
- trace 時の訪問順と追加順
