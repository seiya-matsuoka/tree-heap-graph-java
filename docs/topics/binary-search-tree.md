# 二分探索木

## 1. 概要

二分探索木は、二分木の一種で、**左部分木には小さい値、右部分木には大きい値** を置くルールを持つ木構造。  
このルールにより、木をたどりながら値を探せる。

今回のトピックでは、次の内容を確認する。

- 挿入
- 探索
- 左右の子への分岐
- 中順走査
- レベル順の並び

## 2. 基本の動き

二分探索木では、値を追加するときも探すときも、現在見ているノードの値と比較する。  
比較結果に応じて、左へ進むか右へ進むかを決める。

このトピックでは、次の流れで二分探索木を確認する。

1. 入力値を 1 つずつ木へ追加する
2. 値の大小関係に応じて左 / 右へ進む流れを確認する
3. 追加後の親子関係を確認する
4. target の探索結果を確認する
5. 中順走査で昇順になることを確認する
6. レベル順に見たときの並びも確認する

## 3. 処理の流れ

このトピックでは、次の順で二分探索木の状態を確認する。

1. 先頭値を root として木を作る
2. 残りの値を 1 つずつ比較しながら挿入する
3. target を探し、見つかるまでの流れを確認する
4. 未発見 target も探し、見つからないケースを確認する
5. 木全体の親子関係を整理する
6. 中順走査を行い、値の並びを確認する
7. レベル順の並びを確認する

trace あり実行では、どの値と比較して左 / 右へ進んだかを確認できる。  
そのため、二分探索木がどのような分岐ルールで動いているかを出力から追える。

## 4. 計算量

### 挿入

- 時間計算量: 平均 O(log n)、最悪 O(n)
- 空間計算量: O(1)

### 探索

- 時間計算量: 平均 O(log n)、最悪 O(n)
- 空間計算量: O(1)

### 中順走査・レベル順確認

- 時間計算量: O(n)
- 空間計算量: O(n)

木の形が偏ると、挿入も探索も線形探索に近い重さになる。  
一方、比較的バランスよく分岐していれば、高さが低くなり、少ない比較で目的の値へ近づける。

## 5. メリット / デメリット

### メリット

- 値の大小関係を利用して探索できる
- 中順走査をすると昇順になる
- 配列全体を毎回先頭から見るより、少ない比較で目的値へ近づけることがある

### デメリット

- 木が偏ると効率が悪くなる
- 配列より構造が複雑
- 平衡を保つ仕組みまでは今回は扱わない

## 6. Java実装のポイント

- ノードは `BinarySearchTreeNode` クラスで表現する
- `left` と `right` をそれぞれ別フィールドで持つ
- 追加したい値と現在ノードの値を比較し、左 / 右を選ぶ
- 中順走査は「左 → 自分 → 右」の順でたどる
- runner 側では入力決定、target 決定、結果表示を担当し、学習本体は `algorithms/` に寄せる

## 7. コアとなる処理・重要なコードの見どころ

二分探索木で重要なのは、**値の大小関係に応じて左 / 右へ進む処理** と、**中順走査で値を取り出す処理**。

### 挿入の分岐処理

値を追加するときは、現在ノードと比較し、小さければ左、大きいまたは等しければ右へ進む。

```java
if (valueToInsert < current.value) {
    current = current.left;
} else {
    current = current.right;
}
```

ここで見たいのは、配列の index ではなく、**大小関係で進む方向を決める** 点。

### 探索の分岐処理

探索でも考え方は同じ。  
現在ノードと比較して、左 / 右どちらを見ればよいかを決める。

```java
if (target < current.value) {
    current = current.left;
} else {
    current = current.right;
}
```

この流れにより、先頭から順番に全部見るのではなく、木の高さぶんだけ比較して進める可能性がある。

### 中順走査

中順走査では、左部分木、自分、右部分木の順でたどる。

```java
collectInorderValues(node.getLeft(), values);
values.add(node.getValue());
collectInorderValues(node.getRight(), values);
```

二分探索木では、この順でたどると昇順になる。  
ここが普通の二分木との大きな違いの 1 つ。

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic binary-search-tree --trace
```

### 任意入力

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic binary-search-tree --input 40,20,60,10,30,50,70 --target 30 --trace
```

### size 指定

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic binary-search-tree --size 15 --target 70
```

### 出力で確認する内容

- 入力値
- 探索 target
- 未発見確認用 target
- ルート
- ノード数
- 高さ
- 挿入結果
- 探索結果
- 親子関係
- 中順走査
- レベル順の並び
- trace 時の比較と分岐の流れ
