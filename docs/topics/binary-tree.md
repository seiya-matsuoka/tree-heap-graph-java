# 二分木

## 1. 概要

二分木は、各ノードが **左の子** と **右の子** を最大 1 つずつ持てる木構造。  
一般木より形が制限される分、左右を区別した処理を考えやすい。

今回のトピックでは、次の内容を確認する。

- 左の子 / 右の子
- ルート
- 葉
- 深さ
- 高さ
- レベル順の並び

## 2. 基本の動き

二分木では、各ノードが持てる子は最大 2 つまで。  
そのため、親ノードから見たときに「左へ進む」「右へ進む」という区別がある。

このトピックでは、次の流れで二分木を確認する。

1. 入力配列をレベル順の並びとみなして二分木を作る
2. 各ノードの左の子 / 右の子を確認する
3. 子を持たないノードを葉として確認する
4. ルートからの段数で深さを確認する
5. 一番深い葉までの段数で高さを確認する
6. レベル順に見たときの並びを確認する

## 3. 処理の流れ

このトピックでは、次の順で二分木の状態を確認する。

1. 入力値から二分木ノードを作る
2. `left = 2*i+1`、`right = 2*i+2` の関係で左右の子を接続する
3. 各ノードの親子関係を整理する
4. 葉ノードを集める
5. 各ノードの深さを記録する
6. 木全体の高さを求める
7. レベル順にたどって各レベルの並びを確認する

trace あり実行では、どのノードに左の子 / 右の子を接続したかを確認できる。  
そのため、配列から二分木を作る流れを出力から追える。

## 4. 計算量

### 二分木の構築

- 時間計算量: O(n)
- 空間計算量: O(n)

### 親子関係・葉・深さ・高さの確認

- 時間計算量: O(n)
- 空間計算量: O(n)

### レベル順の確認

- 時間計算量: O(n)
- 空間計算量: O(n)

ノード数を n としたとき、基本的には各ノードを一度ずつたどる処理になる。

## 5. メリット / デメリット

### メリット

- 左右の子を区別できるため、処理のルールを作りやすい
- 二分探索木やヒープなど、後続トピックの土台になる
- 配列表現と対応づけて理解しやすい

### デメリット

- 一般木より制約がある
- 左右を意識して処理を書く必要がある
- 深く偏ると、探索や挿入の効率が悪くなることがある

## 6. Java実装のポイント

- ノードは `BinaryTreeNode` クラスで表現する
- `left` と `right` をそれぞれ別フィールドで持つ
- 入力配列をレベル順の並びとして扱う
- `parentIndex * 2 + 1`、`parentIndex * 2 + 2` で左右の子位置を求める
- runner 側では入力決定と結果表示を担当し、学習本体は `datastructures/` に寄せる

## 7. コアとなる処理・重要なコードの見どころ

二分木で重要なのは、**左右の子を区別して接続する処理** と、**左右へ分かれながら木をたどる処理**。

### 左右の子を接続する処理

今回の実装では、配列の位置から左右の子ノードを求めて接続している。

```java
int leftChildIndex = parentIndex * 2 + 1;
int rightChildIndex = parentIndex * 2 + 2;
```

このルールにより、配列の並びから二分木の形を作れる。  
ここで見たいのは、二分木では「左の子」「右の子」が明確に分かれている点。

### 深さを求める処理

各ノードの深さは、親から子へ進むたびに 1 増やして記録している。

```java
private void collectDepthLines(BinaryTreeNode node, int depth, List<String> lines) {
    if (node == null) {
        return;
    }

    lines.add(node.getValue() + " : depth=" + depth);
    collectDepthLines(node.getLeft(), depth + 1, lines);
    collectDepthLines(node.getRight(), depth + 1, lines);
}
```

ここでは、左部分木と右部分木へ再帰的に進んでいる。  
この流れは、今後の木の走査にもそのままつながる。

### レベル順の並びを確認する処理

レベル順ではキューを使い、上の段から順にノードを見ていく。

```java
Queue<BinaryTreeNode> queue = new ArrayDeque<>();
queue.add(root);
```

この考え方は、後で学ぶ木やグラフの BFS の土台になる。
