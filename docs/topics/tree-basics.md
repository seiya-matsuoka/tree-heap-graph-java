# 木の基礎

## 1. 概要

木は、ノード同士が親子関係でつながるデータ構造。  
先頭の 1 ノードをルートとし、そこから子ノードへ枝分かれしていく。  
配列や連結リストのように一直線ではなく、**階層構造** を持つ点が特徴。

今回のトピックでは、次の内容を確認する。

- ルート
- 親 / 子
- 葉
- 深さ
- 高さ
- レベル順の並び

## 2. 基本の動き

一般木では、1つのノードが複数の子を持てる。  
そのため、あるノードから複数方向へ枝分かれする構造になる。

このトピックでは、次の流れで木を確認する。

1. ルートから親子関係を見る
2. 子を持たないノードを葉として確認する
3. ルートから何本下にあるかで深さを確認する
4. 一番深い葉までの段数で高さを確認する
5. レベル順に見たときの並びを確認する

## 3. 処理の流れ

このトピックでは、次の順で木の状態を確認する。

1. 入力値から一般木を構築する
2. 各ノードについて親子関係を整理する
3. 子を持たないノードを葉として集める
4. ルートから各ノードまでの段数を深さとして記録する
5. 一番深い位置までの段数を高さとして求める
6. 幅優先で木をたどり、各レベルの並びを確認する

trace あり実行では、親子関係をどの順で接続しているかも確認できる。  
そのため、木構造がどのように作られているかを出力から追える。

## 4. 計算量

### 木の構築

- 時間計算量: O(n)
- 空間計算量: O(n)

### 親子関係・葉・深さ・高さの確認

- 時間計算量: O(n)
- 空間計算量: O(n)

### レベル順の確認

- 時間計算量: O(n)
- 空間計算量: O(n)

ノード数を n としたとき、基本的には各ノードを一度ずつたどる処理になる。

## 5. メリット / デメリット

### メリット

- 階層構造を自然に表現できる
- 親子関係を扱う問題と相性がよい
- ファイル構成、カテゴリ構造などをイメージしやすい

### デメリット

- 配列より構造が複雑になりやすい
- どの順でたどるかを意識する必要がある
- 分岐が増えるため、処理の流れを追いづらくなりやすい

## 6. Java実装のポイント

- ノードは `TreeNode` クラスで表現する
- 子ノード一覧は `List<TreeNode>` で保持する
- 親ノード参照も持たせることで、親子関係を説明しやすくする
- 木の構築と、木をたどって情報を集める処理を分ける
- runner 側では入力決定と結果表示を担当し、学習本体は `datastructures/` に寄せる

## 7. コアとなる処理・重要なコードの見どころ

木の基礎で重要なのは、**ノード同士を親子関係でつなぐ処理** と、**木をたどって情報を集める処理**。

### 親子関係を作る処理

`TreeNode#addChild(...)` で、親ノードへ子ノードを追加している。  
ここで「ノードが別のノードを子として持つ」という木の基本構造が作られる。

```java
public void addChild(TreeNode child) {
    child.parent = this;
    children.add(child);
}
```

ここで見たいのは、値を配列の index で管理するのではなく、**ノード同士の参照で構造を作る** 点。

### 深さと高さを求める処理

深さは、ルートから何段下にあるかを表す。  
今回の実装では、親から子へたどるたびに深さを 1 増やしながら記録している。

```java
private void collectDepthLines(TreeNode node, int depth, List<String> lines) {
    lines.add(node.getValue() + " : depth=" + depth);
    for (TreeNode child : node.getChildren()) {
        collectDepthLines(child, depth + 1, lines);
    }
}
```

高さは、そのノードから一番深い葉までの段数。  
子ノード側の高さを再帰で求め、その最大値に 1 を足している。

### レベル順の並びを確認する処理

レベル順の確認ではキューを使う。  
これは後で学ぶ BFS の基本的な考え方にもつながる。

```java
Queue<TreeNode> queue = new ArrayDeque<>();
queue.add(root);
```

このトピックでは、木を上から下へ、左から右へ見る流れを確認することが目的。

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic tree-basics --trace
```

### 任意入力

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic tree-basics --input 10,20,30,40,50,60,70 --trace
```

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic tree-basics --size 10
```

- レベル順の並び
