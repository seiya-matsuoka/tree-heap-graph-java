# ヒープ

## 1. 概要

ヒープは、完全二分木の形を保ちながら、親ノードと子ノードの大小関係を持たせるデータ構造。  
今回のトピックでは、**最大ヒープ** を扱い、配列で表現したときの親子関係、追加、最大値の取り出しを確認する。

今回のトピックでは、次の内容を確認する。

- 完全二分木
- 配列表現
- 親 / 子の index 関係
- 追加
- 最大値の取り出し
- レベル順の並び

## 2. 基本の動き

最大ヒープでは、親ノードは子ノード以上の値を持つ。  
そのため、先頭には常に最大値が来る。

このトピックでは、次の流れでヒープを確認する。

1. 値を 1 つずつ追加する
2. 末尾へ追加した後、親と比較しながら上へ持ち上げる
3. 追加が終わったら、配列全体が最大ヒープになっていることを確認する
4. 最大値を順に取り出し、値が大きい順に出てくることを確認する
5. レベル順の並びと親子関係を確認する

## 3. 処理の流れ

このトピックでは、次の順でヒープの状態を確認する。

1. 入力値を空のヒープへ 1 つずつ追加する
2. 追加した値を親と比較し、必要なら交換して上へ持ち上げる
3. 追加後の内部配列を確認する
4. 配列から親子関係を整理する
5. レベル順の並びを確認する
6. 先頭の最大値を取り出し、末尾要素を root へ移した後で下へ沈める
7. 全要素を取り出し、取り出し順を確認する

trace あり実行では、上方向の比較、下方向の比較、交換の流れを確認できる。  
そのため、最大ヒープ条件をどのように保っているかを出力から追える。

## 4. 計算量

### 追加

- 時間計算量: O(log n)
- 空間計算量: O(1)

### 最大値の取り出し

- 時間計算量: O(log n)
- 空間計算量: O(1)

### ヒープ全体の確認

- 時間計算量: O(n)
- 空間計算量: O(n)

要素数を n としたとき、追加も取り出しも木の高さぶんだけ親子比較を行う。  
完全二分木の高さはおおよそ log n になるため、比較回数もそれに応じて増える。

## 5. メリット / デメリット

### メリット

- 最大値や最小値を効率よく取り出せる
- 配列で表現できるため、親子関係を index で扱える
- PriorityQueue の考え方の土台になる

### デメリット

- 配列全体を昇順 / 降順で持つ構造ではない
- 親子関係の index 計算に慣れるまで分かりにくい
- 途中状態だけを見ると、単純な整列済み配列より直感的ではない

## 6. Java実装のポイント

- 学習用として `MaxHeap` を自前実装している
- 内部では `int[]` を使い、必要に応じて拡張する
- 親 index は `(childIndex - 1) / 2` で求める
- 左の子 index は `2 * parentIndex + 1`、右の子 index は `2 * parentIndex + 2` で求める
- 追加では `siftUp(...)`、取り出しでは `siftDown(...)` を使って最大ヒープ条件を保つ
- runner 側では入力決定と結果表示を担当し、学習本体は `datastructures/` に寄せる

## 7. コアとなる処理・重要なコードの見どころ

ヒープで重要なのは、**追加した値を上へ持ち上げる処理** と、**root に移した値を下へ沈める処理**。

### 上方向へ持ち上げる処理

値を末尾へ追加した後、親と比較して必要なら交換する。

```java
int parentIndex = (currentIndex - 1) / 2;
if (values[parentIndex] >= values[currentIndex]) {
    break;
}
swap(parentIndex, currentIndex);
```

ここで見たいのは、親子の大小関係が崩れていたら交換し、最大ヒープ条件を保つ点。

### 下方向へ沈める処理

最大値を取り出した後は、末尾要素を root へ移す。  
そのままでは親子関係が崩れるため、左右の子と比較しながら下へ沈める。

```java
int leftChildIndex = currentParentIndex * 2 + 1;
int rightChildIndex = currentParentIndex * 2 + 2;
int largestIndex = currentParentIndex;
```

ここで、

- `currentParentIndex` は今比較の基準にしている親ノード位置
- `leftChildIndex` と `rightChildIndex` は左右の子ノード位置
- `largestIndex` は親と子の中で最大値を持つ位置

を表している。  
この比較と交換の繰り返しが、ヒープ本体の処理になる。

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic heap --trace
```

### 任意入力

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic heap --input 50,20,70,10,40,60,30 --trace
```

### size 指定

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic heap --size 10
```

### 出力で確認する内容

- 入力値
- ルート
- ノード数
- 高さ
- 追加結果
- ヒープ配列
- 親子関係
- レベル順の並び
- 最大値の取り出し順
- trace 時の比較と交換の流れ
