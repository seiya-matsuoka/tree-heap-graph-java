# グラフの基礎

## 1. 概要

グラフは、**頂点** と **辺** で関係を表すデータ構造。  
木と違い、1 つの頂点が複数の頂点と自由に結び付けられ、閉路を持つこともある。

今回のトピックでは、次の内容を確認する。

- 頂点
- 辺
- 無向グラフ
- 隣接リスト
- 次数

## 2. 基本の動き

グラフでは、どの頂点とどの頂点がつながっているかを管理する必要がある。  
今回のトピックでは、無向グラフを隣接リストで表現し、各頂点のつながり方を確認する。

このトピックでは、次の流れでグラフを確認する。

1. 入力辺から頂点同士のつながりを作る
2. 各頂点の隣接頂点一覧を確認する
3. 各頂点の次数を確認する
4. trace あり実行では、辺を追加するたびの変化を確認する

## 3. 処理の流れ

このトピックでは、次の順でグラフの状態を確認する。

1. 頂点数を決める
2. 入力された辺を 1 本ずつ読み取る
3. 無向グラフとして両方向へ接続する
4. 隣接リストを作る
5. 各頂点の次数を求める
6. グラフ全体のつながり方を出力する

trace あり実行では、どの辺を追加したか、追加後に各頂点の隣接リストがどう変わったかも確認できる。  
そのため、グラフがどのように構築されているかを出力から追える。

## 4. 計算量

### 隣接リストの構築

- 時間計算量: O(V + E)
- 空間計算量: O(V + E)

### 隣接リスト・次数の確認

- 時間計算量: O(V + E)
- 空間計算量: O(V)

頂点数を V、辺数を E としたとき、基本的には各辺を一度ずつ追加し、各頂点を一度ずつ確認する処理になる。

## 5. メリット / デメリット

### メリット

- 頂点同士の複雑なつながりを表現できる
- 木では表しづらい関係も表現できる
- 隣接リストを使うと、つながっている相手を確認しやすい

### デメリット

- 配列や木より構造が複雑になりやすい
- どの頂点からどこへ進めるかを常に意識する必要がある
- 辺の向きや有無で表現が変わるため、最初は混乱しやすい

## 6. Java実装のポイント

- 頂点ごとのつながりは `List<List<Integer>>` で表現する
- 外側の `List` が各頂点を表し、内側の `List` が隣接頂点一覧を表す
- 無向グラフでは、`u-v` を追加するとき `u -> v` と `v -> u` の両方を追加する
- runner 側では入力決定と結果表示を担当し、学習本体は `algorithms/` に寄せる

## 7. コアとなる処理・重要なコードの見どころ

グラフの基礎で重要なのは、**辺を追加して隣接リストを作る処理** と、**隣接リストから各頂点のつながりを確認する処理**。

### 隣接リストを作る処理

今回の実装では、無向グラフとして両方向へ辺を追加している。

```java
adjacencyList.get(fromVertex).add(toVertex);
adjacencyList.get(toVertex).add(fromVertex);
```

ここで見たいのは、グラフでは「値を並べる」のではなく、**どの頂点とどの頂点がつながっているか** を記録する点。  
木のように親子へ限定せず、より自由な関係を表している。

### 次数を求める処理

各頂点の次数は、その頂点の隣接リストの長さを見ることで求められる。

```java
lines.add(vertex + " : degree=" + adjacencyList.get(vertex).size());
```

このトピックでは、グラフの基本として「つながりの一覧」と「何本つながっているか」を確認することが目的。

## 8. 実行例

### 基本実行

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic graph-basics --trace
```

### 任意入力

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic graph-basics --input 0-1,0-2,1-3,2-4,4-5 --trace
```

### size 指定

```bash
java -cp out io.github.seiya_matsuoka.treeheapgraph.App --topic graph-basics --size 10
```

### 出力で確認する内容

- 入力辺
- 頂点数
- 辺数
- 隣接リスト
- 各頂点の次数
- trace 時の辺追加順
